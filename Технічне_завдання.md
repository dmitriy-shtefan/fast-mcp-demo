# Технічне завдання: MCP-сервер аналітики продажів

## 1. Мета

Реалізувати демонстраційний MCP-сервер на Python, який надає AI-агенту безпечний доступ лише для читання до локальних даних продажів у CSV-форматі та інструменти для їх аналізу.

Сервер має підтримувати локальний запуск через `stdio` та HTTP-режим для розгортання.

## 2. Технології та склад проєкту

- Python 3.
- Бібліотека `fastmcp`.
- Веб-компоненти FastMCP / Starlette для HTTP health check.
- Джерело даних: локальний CSV-файл.
- Основний модуль: `sales_mcp_server.py`.
- Демонстраційні дані: `sales_data.csv`.
- Залежності: `requirements.txt`.
- Конфігурація HTTP-запуску: `fastmcp.json`.

## 3. Дані

### 3.1. Розташування та доступ

За замовчуванням сервер читає `sales_data.csv` з директорії, де розташований модуль сервера. Інструменти можуть приймати параметр `file_name` для іншого CSV-файлу, але лише в межах цієї директорії.

### 3.2. Обов’язкова схема CSV

| Поле | Тип | Призначення |
| --- | --- | --- |
| `order_id` | string | Ідентифікатор замовлення |
| `date` | ISO date | Дата замовлення у форматі `YYYY-MM-DD` |
| `region` | string | Регіон продажу |
| `channel` | string | Канал продажу |
| `product` | string | Назва продукту |
| `category` | string | Категорія продукту |
| `quantity` | integer | Кількість одиниць |
| `unit_price` | float | Ціна за одиницю, UAH |
| `cost_per_unit` | float | Собівартість одиниці, UAH |
| `customer_segment` | string | Сегмент клієнта |

Під час завантаження для кожного рядка обчислюються:

- `revenue = quantity × unit_price`;
- `gross_profit = quantity × (unit_price - cost_per_unit)`.

За відсутності хоча б однієї обов’язкової колонки сервер має повертати помилку з переліком відсутніх полів.

## 4. Функціональні вимоги

### 4.1. MCP tools

#### `load_sales_summary`

Параметри: `file_name: str = "sales_data.csv"`.

Повертає зведення за всіма замовленнями:

- назву файлу;
- початкову й кінцеву дати періоду;
- кількість замовлень;
- продані одиниці;
- виручку та валовий прибуток у UAH, округлені до 2 знаків;
- валову маржу у відсотках;
- середню вартість замовлення;
- кількість унікальних продуктів;
- виручку за регіонами й каналами, відсортовану за спаданням.

Для порожнього файлу повертає назву файлу, `orders: 0` та повідомлення про порожній CSV.

#### `top_products`

Параметри:

- `file_name: str = "sales_data.csv"`;
- `limit: int = 5` — від 1 до 20 включно;
- `metric: "revenue" | "gross_profit" | "quantity" = "revenue"`.

Групує дані за продуктом, сортує їх за обраною метрикою та повертає не більше `limit` записів. Для кожного продукту повертаються назва, кількість замовлень, кількість одиниць, виручка, валовий прибуток і валова маржа.

Якщо `limit` поза діапазоном 1–20, має виникати `ValueError`.

#### `monthly_sales_report`

Параметри: `file_name: str = "sales_data.csv"`.

Групує дані за місяцем у форматі `YYYY-MM` і повертає хронологічний список з кількістю замовлень, проданими одиницями, виручкою, валовим прибутком і валовою маржею.

### 4.2. MCP resources

- `sales://sample-csv` — повертає текст демонстраційного CSV-файлу.
- `sales://business-questions` — повертає перелік прикладів бізнес-запитань українською мовою.

### 4.3. MCP prompt

`sales_report_prompt(company_context: str = "B2B SaaS компанія")` має повертати україномовний шаблон для короткого Markdown-звіту. Шаблон вимагає executive summary, ключові метрики, топ продуктів, регіони/канали, місячну динаміку та 2–3 рекомендації. Він повинен забороняти вигадувати дані поза MCP tools/resources.

### 4.4. HTTP endpoint

Сервер має надавати неавторизований endpoint `GET /health`, який повертає JSON:

```json
{"status": "ok", "service": "sales-analytics-mcp"}
```

## 5. Нефункціональні вимоги та безпека

- Сервер працює лише на читання та не змінює файли.
- Заборонити абсолютні шляхи в `file_name`.
- Заборонити вихід з директорії застосунку через `..`, symbolic links або інші шляхи.
- Дозволити лише файли з розширенням `.csv` без урахування регістру.
- Для відсутнього файлу повертати `FileNotFoundError`.
- Грошові значення округлювати до двох десяткових знаків; відсотки — також до двох.
- Якщо виручка дорівнює нулю, валова маржа має дорівнювати `0`, без ділення на нуль.
- Сервер не використовує зовнішню мережу, API-ключі або базу даних.

## 6. Режими запуску

### MCP через stdio

```bash
python3 sales_mcp_server.py
```

### MCP через HTTP

```bash
python3 sales_mcp_server.py --http
```

HTTP-режим також має вмикатися, якщо `MCP_TRANSPORT` має значення `http` або `streamable-http`, або задано змінну середовища `PORT`. Значення за замовчуванням: `HOST=0.0.0.0`, `PORT=8000`. Транспорт — stateless HTTP.

## 7. Критерії приймання

1. `load_sales_summary` повертає всі метрики з розділу 4.1 та коректно обробляє порожній CSV.
2. `top_products` підтримує три метрики, правильне сортування та відхиляє `limit < 1` і `limit > 20`.
3. `monthly_sales_report` повертає місяці в хронологічному порядку.
4. Некоректна схема CSV, неіснуючий файл, абсолютний шлях, шлях поза директорією та файл не-CSV спричиняють відповідні помилки.
5. `GET /health` повертає HTTP 200 і визначений JSON.
6. `fastmcp inspect sales_mcp_server.py:mcp` виявляє tools, resources і prompt.
7. Під час виконання не створюються й не змінюються файли даних.
